# 任务调度器

> date: 2020-03-22<br>
> tag : greedy

---
## 题目描述：
给你一个用字符数组 tasks 表示的 CPU 需要执行的任务列表。其中每个字母表示一种不同种类的任务。任务可以以任意顺序执行，并且每个任务都可以在 1 个单位时间内执行完。在任何一个单位时间，CPU 可以完成一个任务，或者处于待命状态。

然而，两个 相同种类 的任务之间必须有长度为整数 n 的冷却时间，因此至少有连续 n 个单位时间内 CPU 在执行不同的任务，或者在待命状态。

你需要计算完成所有任务所需要的 最短时间 。

示例 1：
```
输入：tasks = ["A","A","A","B","B","B"], n = 2
输出：8
解释：A -> B -> (待命) -> A -> B -> (待命) -> A -> B
     在本示例中，两个相同类型任务之间必须间隔长度为 n = 2 的冷却时间，而执行一个任务只需要一个单位时间，所以中间出现了（待命）状态。 
```

1 <= task.length <= 10^4
tasks[i] 是大写英文字母
n 的取值范围为 [0, 100]

---
## 题目解答：
首先，我们考虑数量最多的那个任务。不妨设为A，其数量是N个。设冷却时间为t。
对于整个时间进度来说，A必须吃掉的一部分时间就是A的执行时间加上冷却时间。也就是：1*(N-1)*(t+1)。
我们现在需要做的事情就是，把A的冷却时间尽量填上。如果冷却时间可以填满，后面剩余的任务，没有一个是重复的（不然A就不是数量最多的）,换一句话讲，除了A以外的任务，经过了1*(N-1)*(t+1)时间之后，要么还剩一个，要么一个都不剩。那么对于这些已经没有重叠的任务。时间等于任务总数。

最后的时间，等于：
```
max(len, count_max + (n + 1) * (record[0] - 1))
```
