# 0x02.石子堆问题

> date: 2020-03-27<br>
> tag : dynamic programming 

---

## 问题描述：
有N堆石子，现要将石子有序的合并成一堆，规定如下：每次只能移动相邻的2堆石子合并，合并花费为新合成的一堆石子的数量。求将这N堆石子合并成一堆的总花费最小（或最大）。

---
## 问题解答：
此问题的关键点在于合并的石子堆必须是相邻的，这就限制了我们无法使用一般的贪心算法。<br>
依然采取dp的思路。<br>
$dp[i][k]$表示从i到k进行合并所得到的最优解（最低消耗）<br>
$$dp[i][k] =
\begin{cases}
0 &\text{i = k}\\
min\{dp[i][mid],dp[mid+1][k]\} + sum[i][k] &\text{i < k}
\end{cases}$$
从i合并到k，可以等价于，在i和k中间找一个分界点，分界点左边合成堆，右边合成一堆，最后一次合并的花费就是左右两大堆合并的代价。
